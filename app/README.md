# app/

Streamlit application — the deployed clinical AI demo.

## Files

| File | Purpose |
|---|---|
| `app.py` | Main Streamlit entry point — all UI sections |
| `model.py` | CNN architecture definition + `load_model()` |
| `pipeline.py` | NIfTI → slices → quality filter → inference → subject aggregation |
| `xai.py` | Grad-CAM + LIME + MC-Dropout uncertainty |
| `anatomy.py` | Heuristic anatomical region labelling from GradCAM CoM |
| `narrator.py` | LLM narrative generation via Claude Haiku API |
| `phenotypic.py` | ABIDE-I phenotypic CSV loader + site performance mapping |
| `report.py` | PDF clinical report generation (ReportLab) |
| `weights/` | Model weights — see `weights/README.md` |
| `demo_subjects/` | NIfTI demo files — see `demo_subjects/README.md` |

## Running

```bash
# From repo root
pip install -r requirements.txt

# Set optional LLM narrative API key
export ANTHROPIC_API_KEY=sk-ant-...   # Linux/Mac
$env:ANTHROPIC_API_KEY="sk-ant-..."  # Windows PowerShell

streamlit run app/app.py
```

Then open http://localhost:8501

## Architecture note

The app is split into two clear stages to prevent Streamlit rerun oscillation:

**Stage 1 — Computation** (runs only when input changes):
- Pipeline, GradCAM, MC-Dropout, XAI figures → PNG bytes, LLM narrative, PDF all computed here
- Results stored in `st.session_state` in a single batch write at the end
- No `st.write` / `st.markdown` calls here

**Stage 2 — Display** (runs on every rerun, zero computation):
- Reads exclusively from `st.session_state`
- Uses `st.image(png_bytes)` from bytes — never `st.pyplot()` — to avoid rerender loops
- PDF download button serves pre-generated bytes instantly

This pattern ensures the UI is stable on both local and Hugging Face Spaces.
